# Catificator — Good Enough for Now

Tracked compromises with rationale and future direction.

- **No auth** — future: wallet-based identity
- **No validator healthchecks** — future: heartbeat/TTL
- **Result images served from validator URL directly** — future: copy to faci S3
- **All in-memory** — future: database
- **Random routing** — future: stake-weighted, performance-based
- **No job timeouts** — future: faci-side deadline
- **No image validation/size limits**
- **No job cancellation**
- **No observability beyond logging**
